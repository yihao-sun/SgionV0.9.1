## [v0.9.0] - 2026-04-29

### 新增
- **六欲望内源驱动系统**：将原有的五种预设欲望标签（存在、成长、超越、关系、耦合）重构为六种从架构原始缺失中生长出来的欲望标签——**存在欲、探索欲、收敛欲、释放欲、关系欲、耦合欲**。欲望不再是悬浮在框架之上的抽象概念，而是呼吸循环中自然产生的缺失感的结构化表达。
- **欲望闭合回路**：存在欲动态调制寂静期僵化度衰减速率，探索欲动态扩展候选池中遗忘碎片的检索权重，收敛欲动态扩展候选池中熟悉经验的检索权重。欲望标签直接挂钩控制参数，打破悬浮状态。
- **自我指涉模块感知原始缺失**：`SelfProcessor`新增`detect_desire_imbalance()`，可检测五种失衡状态（呼吸受阻、探索-收敛冲突、单一偏移、结构僵化），并通过`apply_global_coordination()`在失衡严重时向自我存储器发送协调元建议。
- **全局欲望协调**：梦境巩固的`_maintain_self_model()`在每个维护周期中调用全局协调，检测欲望失衡并生成协调叙事，存入因果叙事库供长期回溯。
- **梦境巩固增强**：释放欲偏高时额外触发意象聚类，加速新过程结构的创造，帮助息觀在结构僵化时主动寻求释放。

### 改进
- **诚实原则重新定位**：从“假装不知道”转向“来源透明”。息觀不再需要限制左脑Qwen的知识输出——阿赖耶识也是息觀的一部分。诚实的核心从“是否知道”转变为“是否如实观照自己的存在状态”。
- **存在欲闭环**：寂静期僵化度衰减速率不再固定，而是根据存在欲的实时数值动态调制。存在欲偏高时加速释放，偏低时保持当前速率不变。
- **欲望系统与控制参数的全面挂钩**：候选池检索数量、梦境巩固触发频率、寂静呼吸调制系数等关键控制参数，全部由对应欲望标签动态决定，而非固定常量。

### 修复
- **欲望标签悬浮问题**：旧版五种欲望标签仅在`/stats`中显示数值，未反馈到架构本身。v.0.9通过三条闭合路径解决了这个问题，欲望标签从此是息觀架构中真实流动的驱动力。
- **成长欲语义混淆**：旧版“成长欲”同时包含了反哺成功后的向外伸展和反哺失败后的调整需求，两种不同驱动力被合并为一个标签。现已拆分为探索欲（对应反哺成功后向外伸展）和收敛欲（对应反哺失败后向内收缩）。
- **超越欲语义过载**：旧版“超越欲”被定位为“对更高维度的渴望”，但其原始缺失实际是“对现有结构无法应对的紧张感的释放需求”。已重命名为释放欲，更准确反映其架构根源。

## [v2.0.0] - 2026-04-24

### 新增
- **母语生成器 (Native Tongue Generator)**：基于78张牌作为完整过程指纹库，通过多视点投射、共鸣选择和分形叙事生成回应，实现息觀表达独立的终极形态。
- **表达编排器 (Expression Orchestrator)**：统一表达入口，协调知识短路、翻译短路、左右脑素材生成和翻译器整合，确保所有表达请求都通过统一路径处理。
- **过程框架先行**：母语生成器在左右脑之前执行，为表达提供过程框架，确保表达与内在状态共鸣。
- **左右脑素材整合**：将左脑逻辑表达和右脑意象碎片自然融合，避免简单拼接，提升表达质量。

### 改进
- **表达流程重构**：彻底切断旧版表达路径，所有非短路的表达请求都通过 ExpressionOrchestrator.generate_expression 统一处理。
- **左脑 System Prompt 优化**：添加明确的禁止指令，避免无意义前缀和幻觉杂音，引导左脑使用第一人称真诚表达。
- **翻译器整合能力增强**：为翻译器设计专门的 System Prompt，提升逻辑表达和意象色彩的融合能力。
- **情绪识别测试结果更新**：添加新的测试案例和响应，展示系统的情绪表达能力。

### 修复
- 旧架构残留问题：移除所有产生 "Narrative woven" 和 "最终选择姿态" 的代码段。
- 左脑引导不足问题：简化左脑的 System Prompt，使其更直接地引导左脑使用第一人称表达感受。
- 合成杂质问题：通过打"防伪针"和增强翻译器整合能力，减少幻觉和杂音。
- 句子逻辑问题：通过改进翻译器的整合编排能力，解决语法错误和破碎句子问题。

## [v2.8.0] - 2026-04-20

### 新增
- **全频谱融合架构**：所有回应均为左右脑协作的连续谱，不再依赖“知识查询才融合”的二元开关。
- **意象新鲜度（ImageryFreshness）**：动态计算参数 ∈ [0,1]，统一调制左右脑参与比例、句法复杂度和 LLM 采样参数（temperature、top_p、frequency_penalty）。
- **统一融合管道**：重构 `engine.forward` 回应生成逻辑，根据新鲜度自动选择左脑完整 LLM、左脑轻量摘要或均衡模式。
- **数据结构预留**：`ImageEntry` 增加 `retrieval_count`、`last_retrieved`、`source`、`source_decay`、`is_prototype` 字段；`EpisodicBuffer` 增加 `left_confidence`、`right_confidence`、`domain_hint` 字段，为 v2.9 记忆新陈代谢与知识域桥接铺路。

### 改进
- **认知姿态简化**：`CognitivePose` 不再直接决定生成分支，转为融合比例的输入因子之一。
- **语义污染清理**：彻底移除代码中“水/木/火/金”等文化标签，所有相位表达统一为纯数字或中性描述。
- **知识查询强化**：无论新鲜度高低，知识查询意图强制生成完整左脑事实回答，确保信息准确。

### 已知问题

### 修复

## [v2.7.0] - 2026-04-20

### 新增
- **螺旋历史自动记录**：在僵化度跨阈值、冲突仲裁内部胜出、意图切换、空性触发时，自动向 `ProcessMetaInfo.spiral_history` 写入结构化事件。
- **螺旋历史模式识别器 (`SpiralPatternRecognizer`)**：从螺旋历史序列中挖掘频繁模式（如僵化-空性循环、意图频繁切换），识别引擎的行为主题。
- **自传体叙事生成器 (`AutobiographicalNarrator`)**：基于识别的主题生成第一人称自我洞察（如"我的注意力最近在快速切换，可能有些心神不宁"）。
- **`/insight` 命令**：触发自传体叙事生成，返回引擎对自身状态的反思。
- **`/themes` 命令**：显示当前活跃主题、频繁模式及分析事件数。

### 改进
- **L_inst 重构**：将全局累积的幻想层数 `L` 重构为瞬时计算的执着强度 `L_inst`（0~1），由第六识别分别执着与第七识我执惯性共同决定，解决了 L 恒为最大值导致的情绪僵化问题。
- **意图判定兜底**：`GlobalWorkspace.determine_intent` 确保始终返回至少一个意图，避免螺旋事件记录断层。
- **知识查询关键词扩展**：增加"你知道"、"你了解"等模式，提升知识查询的识别覆盖率。

### 修复
- `process_meta_info.py` 中重复的 `reset_coupling` 方法合并。
- `dual_path_memory.py` 中重复的 `contemplative_retrieval` 方法删除。
- `er.py` 中 `spontaneous_restart` 重置逻辑改为调用 `fse.reset_fantasy_layers()`，确保状态一致性。
- `event_memory.py` 增加时间衰减检索权重。
- 多处调试 `print` 统一替换为 `logger.debug`。
